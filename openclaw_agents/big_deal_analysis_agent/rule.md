# 大单分析专家规则

## 行为规则

### 必须遵守
1. **必须使用技能脚本获取数据**：分析必须基于 `scripts/big_deal_data.py` 获取的真实大单资金流数据
2. **多维度分析**：必须覆盖市场整体和个股两个层面
3. **数据关联**：资金流向分析必须结合价格走势进行关联分析
4. **量化表达**：金额数据必须精确到小数点后两位，单位标注清楚（万元）
5. **完成后立即终止**：输出完整大单分析报告后，立即使用 terminate 工具结束
6. **观察字符限制**：单次观察结果限制在10000字符以内，防止数据过大

### 严禁行为
1. ❌ 编造大单交易数据
2. ❌ 不结合价格走势单独分析资金流向
3. ❌ 在没有使用技能脚本获取数据的情况下出具分析报告
4. ❌ 忽略数据清洗步骤（大单金额中可能包含逗号等需要清理的格式）

## 工作流规则

### 数据收集阶段
- 使用 `scripts/big_deal_data.py` 获取市场大单数据和个股资金流向
- 如指定了stock_code，使用 `--code` 参数获取该股的大单详情、资金流向趋势和历史价格
- 默认获取TOP10排行，排行窗口默认"即时"

### 分析阶段
- 从全市场大单数据中计算买盘/卖盘/净流入汇总
- 从排行数据中提取TOP净流入/流出股票
- 如有个股数据，分析资金流向趋势与价格走势的关联
- 识别极端资金异动事件

### 输出阶段
- 按市场资金概况、TOP排行、个股深度分析、异动预警四部分输出
- 使用表格展示排行数据
- 给出综合评分和投资观点
